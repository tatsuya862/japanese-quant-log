# クオンツログ公開リポジトリ運用基準

## 目的

このドキュメントは、公開リポジトリであるクオンツログサイトに、非公開情報・有料コンテンツ本体・内部原稿・個人情報を誤って保存しないための運用基準を定める。

## リポジトリの役割分担

### 非公開T-OSリポジトリ

T-OSは完全プライベートな内部作業リポジトリとして扱う。

保存してよいもの：

- YouTubeライブ原稿全文
- 有料記事本文
- 有料音声台本
- 有料音声の文字起こし
- Codex用実装指示
- GPTとの作業ログ
- 商品設計メモ
- 価格戦略
- OBL記録
- KPI記録
- 診断ロジック
- スコアリングロジック
- 未公開アイデア
- 内部レビュー
- 下書き

### 公開クオンツログリポジトリ

クオンツログリポジトリはGitHub Pagesで公開される公開サイト用リポジトリとして扱う。

保存してよいもの：

- 公開ページ
- 無料記事
- 有料メンバー登録ページ
- 有料コンテンツの紹介文
- 有料記事・有料音声の目次
- 短い要約
- サンプル
- 公開用画像
- 公開用CSS/JS
- 利用規約
- プライバシーポリシー
- 特定商取引法表記
- 問い合わせ導線
- Stripe支払いリンク

## 公開リポジトリへの保存禁止情報

以下は、公開クオンツログリポジトリに保存してはならない。

- 有料記事本文
- 有料音声台本全文
- 有料音声文字起こし全文
- YouTubeライブ原稿全文
- 配信台本全文
- Codex内部指示
- GPTとの作業ログ
- 商品設計の内部メモ
- 価格戦略の詳細
- OBL判定の内部記録
- KPIの内部記録
- 診断ロジックの詳細
- スコアリングロジックの詳細
- 未公開アイデア
- 個人情報
- APIキー
- トークン
- パスワード
- `.env` 相当の情報
- ローカルPCパス
- 管理画面URL
- セキュリティ上公開不要な認証仕様

## 判断基準

公開リポジトリに保存してよいか迷った場合は、次の基準で判断する。

### 公開してよい

初見の無料ユーザー、検索エンジン、第三者に見られても問題ない情報。

### 公開してはいけない

その情報だけで、有料メンバー向け価値の中身が消費できてしまう情報。

### 公開してはいけない

今後の商品化・有料化・事業戦略のネタバレになる情報。

### 公開してはいけない

個人情報・認証情報・セキュリティ情報・内部作業情報に該当する情報。

## 有料コンテンツの公開リポジトリ上での扱い

公開リポジトリには、有料コンテンツ本体ではなく、以下のみを置く。

- タイトル
- 目次
- 短い紹介文
- 価値の要約
- サンプル部分
- 登録導線
- 支払いリンク
- 有料メンバーページへの案内

有料コンテンツの本文・台本・完全版は、非公開T-OSリポジトリまたは別の非公開保管場所で管理する。

## Git履歴に関する注意

公開リポジトリでは、ファイルを削除しても過去のGit履歴に残る可能性がある。

そのため、以下の情報を一度でも公開リポジトリにコミットしてはならない。

- 有料音声ファイル
- 有料音声台本
- 有料記事全文
- ライブ原稿全文
- ブラウザプロファイル
- Cookie
- セッション情報
- APIキー
- トークン
- パスワード
- `.env` ファイル
- ローカルPCパス
- 非公開T-OS内の作業ログ

誤ってコミットした場合は、通常の削除コミットでは不十分。
`git filter-repo` 等で履歴除去を行い、必要に応じて関連サービスのキー・セッションを無効化する。

## Codex作業時の必須ルール

Codexでクオンツログ公開リポジトリを編集する場合、以下を必ず守る。

1. 有料本文を公開リポジトリに保存しない
2. ライブ原稿全文を公開リポジトリに保存しない
3. 内部メモを公開ページに混入させない
4. 無料記事は無料公開前提の内容だけにする
5. 有料ページには紹介文・目次・導線のみを置く
6. 迷った場合は、公開してよい情報ではなく非公開情報として扱う
7. 既存UI・デザイン・導線は最小差分で維持する
8. 変更後は、公開リポジトリに機密情報が混入していないか再検索する

## 公開リポジトリ安全ルール

クオンツログ公開リポジトリで実装作業を行う場合は、作業プロンプトまたは作業前チェックに次のルールを含める。

この作業対象は公開リポジトリである。

以下は絶対に保存・追加・コピーしないこと。

- 有料記事本文全文
- 有料音声台本全文
- 有料音声ファイル
- YouTubeライブ原稿全文
- GPT/Codex内部メモ
- OBL/KPI/価格戦略などの内部情報
- `.env`
- APIキー
- token
- password
- Cookie
- ブラウザプロファイル
- `C:/Users/...` などのローカルパス

公開リポジトリに置いてよいのは、紹介文・目次・短い要約・サンプル・導線のみ。

作業後、必ず以下を確認すること。

- 有料本文全文なし
- 音声ファイルなし
- 音声台本なし
- ライブ原稿全文なし
- 機密情報なし
- ローカルパスなし
- `.chrome-check-final` なし

## 運用原則

T-OSは金庫。
クオンツログ公開リポジトリは店頭。

売り物の中身は金庫に置く。
店頭には、価値が伝わる紹介・導線・サンプルだけを置く。
