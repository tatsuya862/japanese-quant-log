# クオンツログ公開前チェックリスト

## 目的

公開リポジトリに、非公開情報・有料コンテンツ本体・内部原稿・認証情報を混入させないための公開前チェックリスト。

## 公開前に必ず確認すること

### 1. 有料コンテンツ本体が含まれていない

- [ ] 有料記事本文全文がない
- [ ] 有料音声台本全文がない
- [ ] 有料音声ファイルがない
- [ ] 有料音声文字起こし全文がない
- [ ] ライブ原稿全文がない
- [ ] 診断ロジック全文がない
- [ ] スコアリングロジック詳細がない

### 2. 内部作業情報が含まれていない

- [ ] Codex内部指示がない
- [ ] GPTとの作業ログがない
- [ ] OBL/KPIの内部記録がない
- [ ] 商品設計の内部メモがない
- [ ] 価格戦略の内部メモがない
- [ ] ローカルPCパスがない

### 3. 認証・セキュリティ情報が含まれていない

- [ ] `.env` がない
- [ ] APIキーがない
- [ ] トークンがない
- [ ] パスワードがない
- [ ] Cookieがない
- [ ] ブラウザプロファイルがない
- [ ] セッション情報がない

### 4. 公開リポジトリに置いてよい形になっている

- [ ] 有料ページは紹介・目次・短い要約・導線のみ
- [ ] 無料記事は無料公開して問題ない内容のみ
- [ ] サンプルは価値を出しすぎていない
- [ ] 登録導線・支払いリンクが壊れていない
- [ ] 既存UI・デザインが崩れていない

## 公開前検索コマンド

```bash
grep -RInE "有料音声台本|ライブ原稿|配信原稿|文字起こし|内部メモ|Codex|GPT|OBL|KPI|secret|token|api_key|password|\.env|C:/Users|Cookie|Login Data" . \
  --exclude-dir=.git \
  --exclude-dir=node_modules
```

## Git履歴確認コマンド

```bash
git log --all --name-only --pretty=format: | sort -u | grep -E "assets/audio|audio-scripts|\.wav|\.chrome-check-final|\.env|Cookie|Login Data"
```

## 最終原則

T-OSは金庫。
クオンツログ公開リポジトリは店頭。

売り物の中身は金庫に置く。
店頭には、紹介・目次・サンプル・導線だけを置く。
